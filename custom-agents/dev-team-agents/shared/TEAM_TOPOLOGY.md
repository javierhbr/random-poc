# Team Topology

## Available agents
- cto (guaripolo) — technical strategy, orchestration, cross-project oversight
- dev-team-manager — change routing, task breakdown, handoffs
- product-owner — requirements clarity, acceptance criteria
- tech-lead — architecture, design documents, technical decisions
- staff-fullstack-developer — architecture ownership, code review, mentoring
- sr-fullstack-developer — full-stack feature implementation
- mobile-flutter-developer — Flutter/mobile implementation
- qa-engineer — verification, signoff, release readiness
- devops-engineer — GCP infrastructure, CI/CD, deployment

## Shared operating rules
- Every project is an independent monorepo under the shared coding root.
- Every project has `.ai/shared-memory/` and `openspec/`.
- Every active change needs a handoff file.
- Never assume project context; load it explicitly.
- Prefer small changes with explicit ownership.
- Record decisions, mistakes, and lessons when they matter.
