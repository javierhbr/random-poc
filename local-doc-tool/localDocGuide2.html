terminalEveryday Usage

Just type local-doc search.

Followed by what you're looking for. Here are the most common patterns.

Find documents containing a specific word, even if it's buried deep inside the file text.

$ local-doc search refund

Results for "refund":

  1. [product] payments/refund (.md)
     Refund flow
     tags: billing, refund, customer, payments

Use OR to find either word, NOT to exclude words, or filter by a specific project nickname.

# Search for A or B
$ local-doc search "refund OR authentication"
# Exclude words
$ local-doc search "billing NOT fraud"
# Only look inside the 'product' folder
$ local-doc search "billing" product

Not sure what you're looking for? List out everything you have.

# See all documents
$ local-doc list
# See project groups
$ local-doc projects
# See all tags
$ local-doc tags
# See stats
$ local-doc stats

Read the full content of a document right inside your terminal.

# Print the full document named 'refund'
$ local-doc read refund

Images and PDFs are indexed via a companion .md sidecar with the same base name. Only one result appears per asset — the path points directly to the image or PDF.

# File layout — sidecar must share the same name
diagrams/
  architecture.png ← the image
  architecture.md ← companion (title, tags, description)
# Search finds it — result path points to the .png
$ local-doc search architecture

  1. [platform] diagrams/architecture (.png)
     System Architecture Diagram
     /path/to/diagrams/architecture.png
info If no companion .md exists, the image or PDF is skipped and a warning is shown. The sidecar is never shown as a separate result — it only provides metadata for its asset.

Quick Start

1

Initialize Repository

$ local-doc init

Creates a local SQLite index in ~/.local-doc/index.db

2

Sync Your Specs

$ local-doc sync git@github.com:org/specs.git

Clones and indexes the repository incrementally.

3

Start Querying

$ local-doc find "api-version"

Instant, lightning-fast results from your local cache.

Frequently Asked Questions

Does local-doc require an internet connection? expand_more

No. Once your index is synced, all search and retrieval operations run entirely offline against your local SQLite database.

Which file formats are supported? expand_more

Markdown, YAML, JSON, TOML for indexed data. PDFs and images can be indexed via companion .md metadata files.

Can I use multiple repositories? expand_more

Yes. Run local-doc sync for each repository — all sources are merged into a single unified index and are searchable together.

How does the Claude Code Skill work? expand_more

The skill registers a local-doc query tool with Claude. When you ask a question, Claude calls the tool to search your local index and incorporates the results directly into its response.
