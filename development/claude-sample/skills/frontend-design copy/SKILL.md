# skill:frontend-design

## Does exactly this

Guides creation of production-grade frontend interfaces with bold aesthetic vision: choose a clear design direction (brutalist, retro, organic, minimalist, maximalist, etc.), then execute with precision across typography, color, motion, composition, and details.

## When to use

- Building a new page, component, or full application
- Interface needs distinctive visual identity (not generic AI defaults)
- User asks for "memorable design", "creative interface", or "beautiful UI"
- You need guidance on aesthetic direction before coding

## Do not use this skill when

- Building standard utility components (buttons, inputs, modals belong in `shadcn-ui` skill)
- Interface is purely functional with no aesthetic requirements
- Refactoring existing design (use with caution — may require redesign, not refresh)

## Steps — in order

1. **Understand context** — Purpose, audience, constraints, technical limits
2. **Choose aesthetic direction** — Pick ONE clear vision (brutalist, retro-futuristic, organic, luxury, playful, editorial, industrial, art deco, soft pastel, etc.)
3. **Commit to it** — Design entire interface coherent with that direction
4. **Execute implementation** — Production-grade HTML/CSS/React/Vue with working functionality
5. **Refine details** — Typography, color, motion, composition, backgrounds all serve the vision

## Done when

- Interface visually distinctive and memorable
- Aesthetic vision is clear and consistent throughout
- All details (fonts, colors, spacing, animations) intentional and cohesive
- Code is production-ready and functional
- No generic AI aesthetics (overused fonts, cliched gradients, predictable layouts)
