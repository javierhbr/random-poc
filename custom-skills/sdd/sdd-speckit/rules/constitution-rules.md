# Constitution Rules

Use these rules when drafting or updating the project constitution.

## Non-negotiables
- Principles must be explicit and testable.
- Prefer short, enforceable statements over vague values.
- Include quality, testing, security, and UX expectations.
- State tradeoffs when they matter.
- Avoid project-specific implementation details unless they are permanent constraints.

## Good examples
- All externally visible behavior must have automated tests.
- Accessibility regressions block release.
- Prefer simple, observable architecture over hidden magic.
