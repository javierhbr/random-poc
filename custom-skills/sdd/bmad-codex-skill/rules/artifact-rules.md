# Artifact quality rules

Every artifact generated with this skill should include:
- purpose / problem statement
- scope and non-goals
- assumptions and constraints
- risks or edge cases
- success criteria / acceptance criteria
- implementation implications

## Tech-spec minimum
- current-state context
- proposed change
- affected systems/files/modules
- approach
- tests
- acceptance criteria

## PRD minimum
- user/problem framing
- goals and non-goals
- functional requirements
- constraints
- measurable success criteria
- delivery notes or story framing

## Architecture minimum
- context and boundaries
- major components
- interfaces and integrations
- data flow or state flow
- tradeoffs
- ADRs when introducing new patterns

## Story minimum
- story goal
- scope boundaries
- implementation notes
- tests
- definition of done
