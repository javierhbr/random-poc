# BMAD overview and philosophy

BMAD Method describes itself as an AI-driven development framework inside the broader BMAD ecosystem. Its core idea is that agents work best when context is built progressively through structured workflows instead of jumping directly into code. The docs frame BMAD as a process that can scale from bug fixes to enterprise systems through specialized agents and guided workflows. citeturn1view0turn1view1turn1view2

## Core principles

- Progressive context beats ad hoc prompting.
- Planning depth should scale with project complexity.
- Specialized agents should be matched to the job.
- Artifacts created in earlier phases should inform later phases.

## Why this matters for a Codex skill

A Codex skill should not just “write docs.” It should:
- route work to the right BMAD track
- produce the right artifact for the current phase
- preserve context for later implementation
- stay grounded in existing project conventions when working brownfield
