# Durable Context Rules

Use these rules when turning current-state findings into a practical Platform
baseline.

## Goal

Create shared context and durable rules that teams can actually use.

## Must do

- keep only durable context in the platform baseline
- turn vague values into explicit and testable rules
- keep the baseline small enough for real adoption
- define role expectations for Platform, Route, and Specify
- encode reusable context in configuration when possible

## Avoid

- stuffing temporary change detail into the platform baseline
- writing ideals that teams cannot apply
- overloading the baseline with too many rules at once

## Required outputs

- draft principles or constitution
- draft reusable project config notes
- role map
- recommended baseline artifacts
