# Current State Rules

Use these rules when documenting the existing platform.

## Goal

Document what is true today before proposing what should change.

## Must do

- inspect the platform as a brownfield environment
- record current conventions, constraints, and artifacts
- identify team interactions and ownership boundaries
- distinguish facts, assumptions, and unknowns
- document evidence in plain language

## Avoid

- inventing a target model before understanding the current model
- replacing existing conventions without first documenting them
- mixing recommendations into the current-state section

## Required outputs

- current-state snapshot
- list of current constraints and conventions
- known unknowns
