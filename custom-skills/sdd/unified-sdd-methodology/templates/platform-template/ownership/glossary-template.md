## How to use this file

- Add terms when the Platform phase defines capabilities or contracts.
- Add terms when Specify uncovers ambiguity that caused confusion in a review.
- All specs, designs, and stories must use only glossary terms for shared concepts.
- If two teams use the same word to mean different things, this glossary resolves it.
- Reference term IDs in `platform-ref.yaml` under
  `alignment_notes.glossary_terms_used`.
